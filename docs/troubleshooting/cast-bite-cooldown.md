# AF-004：抛竿瞬间误判咬钩

状态：已解决

## 问题描述

刚发送 `F` 抛竿后，脚本可能立即检测到“咬钩”，然后立刻收竿。这个误判来自抛竿动作刚发生时的画面变化，而不是真实咬钩。

## 现象

- 日志表现为 `[动作] 抛竿` 后几乎立刻出现 `[检测] 咬钩`。
- 反应时间很短，随后马上进入收竿流程。
- 实际游戏里只是抛竿动作或鱼漂入水过程。

## 根因

- 抛竿后短时间内，鱼漂、鱼影、检测框或画面残留可能短暂满足咬钩条件。
- 旧流程抛竿后立即进入 `fish_touching_float()` 判定，没有冷却窗口。

## 当前解决方案

- 增加 `CAST_BITE_DETECTION_DELAY = 2.0`。
- 抛竿后 2 秒内仍刷新预览窗口，但跳过咬钩判定。
- 2 秒后才允许进入正常咬钩检测。
- 该冷却只作用于抛竿后的检测阶段，不影响收竿动作。

## 涉及代码

- `CAST_BITE_DETECTION_DELAY`
- 主循环中的 `detect_after = start + CAST_BITE_DETECTION_DELAY`
- 检测循环中 `if time.time() < detect_after: continue`

## 后续维护边界

- 不要取消抛竿后的固定检测冷却。
- 如果要调整时长，应优先在 1.5-2.5 秒范围内微调，并观察刚抛竿误判率。
- 不要把抛竿和收竿的 `F` 混用成同一段判定逻辑。
